# POS会計 機能要件

## 取引フロー

```
カート操作 → 支払方法選択 → 支払確定 → レシート表示
```

### 取引作成
- カートに商品を追加（商品ID・数量・単価スナップショット）
- 商品スキャン（バーコード/SKU）で自動検索・追加
- 数量変更・商品削除
- 割引・クーポン適用（取引全体 or 商品単位）

### 税額計算エンジン

| 項目 | 仕様 |
|------|------|
| 標準税率 | 10%（デフォルト） |
| 軽減税率 | 8%（食料品等） |
| 計算方式 | 内税（税込価格から逆算）または外税（税抜価格に加算） |
| 端数処理 | 切り捨て（銭単位BIGINT） |
| インボイス対応 | 適格請求書発行事業者番号をレシートに印字 |
| 税率別集計 | `transaction_tax_summaries` に税率・税額・対象金額を記録 |

### 支払処理

| 支払方法 | 処理 |
|----------|------|
| 現金 | 受取金額入力 → お釣り自動計算（受取 - 税込合計） |
| クレジットカード | 外部決済端末との連携（承認番号記録） |
| QRコード | 外部QR決済APIとの連携（トランザクションID記録） |
| 複合支払 | 複数支払方法の組み合わせ可能 |

### 取引確定
- 取引番号自動採番（店舗別シーケンス: `{store_code}-{YYYYMMDD}-{seq}`）
- `sale.completed` イベント発行（RabbitMQ）
- ステータス: `PENDING` → `COMPLETED`

## レシート生成

- PDF形式（Server-side生成）
- 記載項目: 店舗名・住所・インボイス番号・取引番号・日時・商品明細・税率別集計・支払方法・お釣り
- QRコード埋め込み（取引照会URL）

## 返品・VOID処理

| 操作 | 対象 | 在庫 | イベント |
|------|------|------|----------|
| VOID（当日取消）| 完了済み取引 | 自動戻し | `sale.voided` |
| 返品 | 過去取引 | 自動戻し | `sale.returned` |

- 返品時は元取引IDを参照記録
- 部分返品対応（商品単位）

## オフライン取引

- `client_id`（UUID v4）をクライアントで生成し、取引に付与
- サーバー側で `client_id` による重複排除（冪等性保証）
- オフライン中は IndexedDB にキュー保存
- オンライン復帰後に Background Sync で一括送信

## 受け入れ条件

- [ ] 取引確定から3秒以内にレシートPDFが表示される
- [ ] 同一 `client_id` で2回送信しても取引は1件のみ作成される
- [ ] 税率別集計が `transaction_tax_summaries` に正確に記録される
- [ ] お釣り計算が正確（受取金額 - 税込合計）
