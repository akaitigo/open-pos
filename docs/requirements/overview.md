# open-pos プロジェクト概要

## ビジョン

あらゆる規模の小売店・飲食店が即日導入できる、クラウドネイティブかつオフライン対応の汎用POSシステム。

## 目標

| 目標 | 詳細 |
|------|------|
| マルチテナント | 単一インフラで複数テナント（組織）を安全に分離運用 |
| オフライン継続 | ネットワーク断絶時もPOS会計を止めない |
| インボイス対応 | 適格請求書発行事業者番号・税率別集計に対応 |
| スケーラビリティ | テナント数・端末数の増加に対してサービス単位でスケール |

## 対象ユーザー

- **店舗オーナー (OWNER)**: テナント管理・スタッフ管理・売上分析
- **店長 (MANAGER)**: 商品管理・在庫管理・発注管理
- **レジ担当 (CASHIER)**: POS会計・返品処理

## 主要機能一覧

### POS会計
- カート操作・商品スキャン
- 税率別（10%/8%）自動計算・インボイス対応
- 現金・クレジットカード・QRコード決済
- レシート生成（PDF）
- 返品・VOID処理
- オフライン取引（Background Sync）

### 商品管理
- 商品CRUD（バーコード/SKU対応）
- カテゴリ階層管理
- 割引・クーポン定義

### 在庫管理
- リアルタイム在庫追跡（会計連動自動減算）
- 在庫アラート
- 発注管理（DRAFT→ORDERED→RECEIVED）

### 店舗・スタッフ管理
- 組織・店舗・端末の階層管理
- スタッフPIN認証（端末切替）
- ロール別権限制御

### 売上分析
- 日次・商品別・時間帯別集計
- イベント駆動型リアルタイム集計

## 技術スタック概要

- **Frontend**: React / TypeScript / PWA (Vite)
- **Backend**: Kotlin / Quarkus / gRPC
- **DB**: PostgreSQL（スキーマ分離マルチテナント）
- **Cache**: Redis
- **MQ**: RabbitMQ
- **Auth**: ORY Hydra (OAuth2/OIDC + PKCE)
- **Infra**: GCP (Cloud Run / Cloud SQL / Memorystore)

## 金額の扱い

全金額フィールドは **BIGINT（銭単位）** で管理する。
例: 100円 = `10000`（1銭 = 1単位）
詳細は `docs/adr/005-money-as-bigint.md` 参照。
