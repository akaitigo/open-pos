# ADR-005: 金額を BIGINT（銭単位）で管理

## Status
Accepted

## Context
POSシステムでは金額計算の正確性が最重要。浮動小数点（float/double）は10進数を正確に表現できず、丸め誤差が発生する。例: `0.1 + 0.2 = 0.30000000000000004`。

税額計算・割引計算・お釣り計算で1円のズレも許容できない。

## Decision
全ての金額を BIGINT（銭単位）で管理する。

- 100円 = 10000（銭単位）
- DB: `BIGINT` 型
- Proto: `int64` 型
- TypeScript: `number` 型（整数として扱う）
- 表示時のみ円に変換: `amount / 100`

### 計算ルール
- 税額計算: `Math.floor(price * taxRate)`（切り捨て）
- 割引計算: 整数演算のみ
- お釣り: 整数の引き算

## Consequences

### メリット
- 浮動小数点の丸め誤差が完全に排除される
- 整数演算のみで高速
- DB の BIGINT は十分な範囲（約922兆円）
- 将来的な小数点以下の通貨にも対応可能

### デメリット
- 表示時に変換が必要（`formatMoney()` ユーティリティで統一）
- 開発者が銭単位を意識する必要がある
- 外部システム連携時に変換が必要
