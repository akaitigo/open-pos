# LICENSE ファイルを追加する

**ラベル**: `type:docs`, `P1:high`

## 概要

リポジトリに LICENSE ファイルが存在しない。README.md のライセンス欄には「Private」と記載されている。ポートフォリオとして public 化するにあたり、適切なライセンスを設定する必要がある。

## 現状の問題

- `LICENSE` ファイルなし
- `README.md` L52-54: `## ライセンス\n\nPrivate`
- ライセンスなしで public 化すると、デフォルトで「全権利留保」となり、閲覧はできるがフォーク・利用はできない

## 対応内容

### ライセンスの選択

ポートフォリオの目的に応じて以下から選択：

| ライセンス | 用途 |
|-----------|------|
| **MIT** | 最も寛容。自由に利用・改変可能。ポートフォリオに最適 |
| **Apache 2.0** | MIT + 特許権の明示的な付与。企業利用を想定 |
| **GPL 3.0** | 派生物も同じライセンスを要求。OSSコミュニティ向け |
| **閲覧のみ** | カスタムライセンスで閲覧のみ許可（ポートフォリオ用途） |

### 対応手順

1. ルートディレクトリに `LICENSE` ファイルを追加
2. `README.md` のライセンス欄を更新
3. 全 `package.json` の `license` フィールドを更新（現在は未設定）
