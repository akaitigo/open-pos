# Maintainers

`open-pos` is currently maintained as a small, maintainer-led OSS project.

## Current Maintainers

| Maintainer | GitHub | Role | Scope |
|-----------|--------|------|-------|
| Ryusei Nakayama | [@akaitigo](https://github.com/akaitigo) | Owner / Maintainer | Product direction, architecture, releases, triage |

## Maintainer Responsibilities

- Review and merge pull requests
- Triage issues and keep milestones current
- Maintain CI, dependency hygiene, and security response
- Update roadmap, changelog, and release notes when scope changes materially

## Decision Making

- Architectural and release decisions are made by the active maintainer set.
- For now, the repository owner is the final decision maker when tradeoffs are unclear.
- Significant decisions should be documented in `docs/adr/`, linked issues, or [GOVERNANCE.md](GOVERNANCE.md) when the process itself changes.

## Review Expectations

- Best-effort async review cadence
- Security issues are handled according to [SECURITY.md](SECURITY.md)
- Contributors should expect feedback in GitHub issues and pull requests rather than private channels
